Purpose

A web-based internal system for tracking housing support clients, their documents, monthly financial activity, pool fund contributions, and administrative actions with full auditability.

Core Principles

Clients are stable entities

Financial data is time-based (month + year)

Nothing important is overwritten silently

Admins have power, super_admins have override power

Everything is auditable

2. User Roles & Permissions
2.1 Admin

Admins handle day-to-day operations.

Admin can:

Create and update clients

Upload and manage client documents

Enter monthly financial data (within edit window)

Perform bulk updates

View dashboard, reports, and client data

Admin cannot:

Edit locked months (past 30 days)

Delete financial history

Override system locks

2.2 Super Admin

Super admins exist for compliance and corrections.

Super Admin can:

Do everything an admin can

Edit locked months

Correct historical records

Manage admin users

View full audit logs

Override system restrictions
